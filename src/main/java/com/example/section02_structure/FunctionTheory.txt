Функция (или метод) — именованный фрагмент программного кода (подпрограмма), к которому можно обратиться из другого места программы по имени.

Назначение методов:

дать осмысленное название фрагменту кода с целью облегчения понимания кода другими разработчиками.

изолировать код от остальной части программ.

сократить дублирование кода.

Объявление метода позволяет обособить определенный фрагмент кода и описать что метод будет делать.

Вызов метода позволяет использовать метод для решения конкретной задачи.

Структурно любая консольная программа состоит из трех частей:

Считывание исходных данных для задачи.

Обработка исходных данных с целью получения результата.

Отображение результата конечному пользователю на консоль.

При декомпозиции задачи на метод, происходит следующее преобразование:

Считывание исходных данных для задачи -> Параметры метода.

Обработка исходных данных с целью получения результата -> Тело метода.

Отображение результата конечному пользователю -> Результат выполнения метода в виде возвращаемого значения вызывающей программе.

Структура данных стек используется в программировании для вызовов методов и позволяет легко проследить последовательность и места вызовов методов в коде.

При передачи аргументов методов, аргументы могут передавать по значению и по ссылке.

Только переменные переданные по ссылке могут быть изменены в теле методе.

Переменная, которая объявлена в теле метода называется локальной переменной!

Формальные параметр метода также является локальной переменной!

Областью видимости локальной переменной в Java является блок { }.
